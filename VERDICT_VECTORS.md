# Verdict format vectors — encoder/decoder cross-check (Tier 1)

Authoritative **format-only** vectors for the 20-byte v2 verdict frame (`PROTOCOL.md`
§Response). Each row is the parsed fields ⇄ the exact bytes on MISO. They pin the
**byte layout**, decoupled from any classifier logic, so both sides can verify
independently:

- **FPGA side:** the verdict encoder's testbench drives these field values in and
  asserts it emits the listed hex frame (MSB-first, byte 0 first).
- **Pi side:** `decode_verdict()` asserts each hex frame parses back to the listed
  fields (`test_verdict.py`). The frames here were generated by the Pi's `encode_verdict()`,
  the verified inverse of the decoder.

These are **not** golden vectors for the classifier — a header does not map to a verdict
here. That's Tier 2 (`header → verdict`), in `VERDICT_GOLDEN.md`.

## Frame layout (recap)

| Byte | Field | Meaning |
|---:|---|---|
| 0 | magic | `0xA5` = valid verdict; else "no verdict" |
| 1 | stage-hit mask | bit0 bloom · bit1 port-scan · bit2 rate-anomaly |
| 2 | severity | 0 clean / 1 low / 2 med / 3 high |
| 3 | flags | bit0 escalate |
| 4 | seq | `frame_count & 0xFF`, 1-based |
| 5–19 | reserved | 0 |

## Vectors

| # | 20-byte frame (hex) | valid | threats (mask) | severity | escalate | seq |
|---|---------------------|-------|----------------|----------|----------|-----|
| 1 | `a500000001000000000000000000000000000000` | yes | — (0x00) | 0 clean | no | 1 |
| 2 | `a501030102000000000000000000000000000000` | yes | bloom (0x01) | 3 high | yes | 2 |
| 3 | `a502020003000000000000000000000000000000` | yes | port_scan (0x02) | 2 med | no | 3 |
| 4 | `a504010004000000000000000000000000000000` | yes | rate_anomaly (0x04) | 1 low | no | 4 |
| 5 | `a507030105000000000000000000000000000000` | yes | bloom+port_scan+rate (0x07) | 3 high | yes | 5 |
| 6 | `a5000000ff000000000000000000000000000000` | yes | — (0x00) | 0 clean | no | 255 |
| 7 | `a501030100000000000000000000000000000000` | yes | bloom (0x01) | 3 high | yes | 0* |
| 8 | `0000000000000000000000000000000000000000` | **no** | — | — | — | — |

\* Vector 7 is frame **256** (`256 & 0xFF = 0`): `seq=0` is a legal value, not a sentinel —
validity is decided by `magic` alone. Vector 8 is the "no verdict" frame (first transfer /
post-reset): `magic != 0xA5`, so all other fields are ignored.

Vectors 6–8 are the boundary/edge cases (seq max, seq wrap-to-zero, no-verdict) — they catch
off-by-one seq handling and the magic-vs-seq validity confusion.
